Property Account Anti-Pattern
Using "Property Account" record type when properties are not accounts. Causes reporting confusion, automation conflicts, and prevents proper Account hierarchies.
üí° Don't overload objects with record types for different entity types. Properties need their own object.
Contact + Person Account Confusion
Same humans exist in both Contact and Person Account objects. 9,301 records with no single source of truth.
üí° Pick ONE pattern for people records. Person Account or Contact, not both. Merge and migrate.
Property Lead Misuse
A property is not a lead. MLS data represents physical assets, not people expressing interest.
üí° Create separate MLS Lead object for property data. Reserve Lead object for people-based interest signals.
Record Type Proliferation
Account: 8 record types | Lead: 4 record types. When objects need 8+ record types, they're being asked to do too much.
üí° If record types represent different "things" (not just different processes), you need separate objects.
