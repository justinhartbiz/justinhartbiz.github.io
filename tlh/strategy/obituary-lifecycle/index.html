Primary Objective

Close the loop from signal → verified address → sensitivity-safe outreach → measured response.

Pipeline Stages

1 · Beacon — Obituary Signal Capture

Scheduled scans across SD/OC/LA obituary sources. Extract name, city, death date, relatives, and context notes.

2 · Queue — Lead Queue Creation

New records synced into mail queue with status flags and next-action ownership.

3 · Match — Name to Property

Match obituary name/city against property-owner signals and score confidence.

4 · Verify — Address Verification

Deliverability check gates records into verified vs research queues.

5 · Outreach — Postcard Trigger

Verified records receive sympathy-resource postcard with approved front/back assets.

6 · Timing — Sensitivity Cadence

Day 0–14 monitor, Day 14 resource mailer, Day 21+ soft follow-up, Day 30+ service offer.

7 · Feedback — Outcome Logging

Track no-response/return/contact/consult/win and feed scores back into matching rules.

8 · Weekly Ops — Review + Tune

Weekly review: found, matched, verified, sent, response, consult, win.

Decision Gates

  • Gate A: Confidence score high enough? If no → research queue.
  • Gate B: Lob deliverable address? If no → manual correction.
  • Gate C: Timing window clear? If no → hold until cadence date.
  • Gate D: Response captured? If yes → escalate to consult workflow.

Guardrails

  • Bereavement-safe language only
  • No immediate hard pitch
  • DNC/TCPA check before calls/texts
  • Confidence thresholds before send
