Same humans exist in both Contact and Person Account objects. No single source of truth for "who is this person." Duplicates guaranteed.
üí° Lesson: Pick ONE pattern for people records. Person Account or Contact, not both. Merge and migrate.
Data Model Property Lead Misuse (TLH SF)
System: TLH Salesforce Record Type: Property (on Lead object)
A property is not a lead. MLS data represents physical assets, not people expressing interest. This causes lifecycle confusion and automation complexity.
üí° Lesson: Create separate MLS Lead object for property data. Reserve Lead object for people-based interest signals.

System: TLH Salesforce Account: 8 types | Lead: 4 types

üí° Lesson: If record types represent different "things" (not just different processes), you need separate objects.

Key Insights from Failures

üìä TLH Estate Sales ‚Äî Top 3 Fixable Problems
1. "Too Small" (26% of losses) ‚Äî OC and LA have different sizing economics than SD. Market-specific minimums could capture 30+ additional deals annually.

2. "Decided Not to Sell" (25%) ‚Äî Late-funnel dropout indicates commitment isn't being tested early. Add explicit "commitment checkpoint" at day 7.

3. "Stopped Responding" (9%) ‚Äî Ghost rate reducible with multi-channel follow-up and explicit "status check" touchpoints.
üì± DB SMS Campaigns ‚Äî Failure Patterns
1. Audience-Platform Mismatch ‚Äî Left-leaning audiences on WinRed (right-leaning platform) consistently underperform. Political infrastructure matters.

2. Local Candidates, National Lists ‚Äî OH-4, VA-02, and similar hyper-local candidates sent to national lists hit 15-18% unsub rates. Geographic targeting required.

3. CPA vs. Donation Confusion ‚Äî Many "zero donor" campaigns are CPA-focused. Success metrics need explicit definition before launch.
üóÉÔ∏è Data Model Lessons (TLH Salesforce)
1. Don't Overload Objects ‚Äî 8 Account record types = modeling failure. If record types represent different entity types, create separate objects.

2. Single Source of Truth ‚Äî Contact AND Person Account = guaranteed duplicates. Pick one pattern and migrate.

3. Automation Surface Area ‚Äî 183 flows + 43 triggers on confused data model = compounding tech debt. Fix the model before adding more automation.

‚Ä¢ L2 Soft Suppressions (9+ STOPS) ‚Äî 0.41% CTR, 8%+ unsub. These are burned contacts.
‚Ä¢ CPA Acquisitions ‚Äî Consistent sub-1% CTR across multiple clients. Needs hygiene audit.
